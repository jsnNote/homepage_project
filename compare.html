Apache vs Nginx

웹 서버 소프트웨어

웹 서버 소프트웨어의 종류
    • Windows
    • Linux
    • macOS
    • ...
    • 아파치 소프트웨어 재단에서 관리하는 HTTP서 웹 서버 소프트웨어이다.
    • Apache 서버는 다양한 모듈을 제공하고 있어 확장성이 좋다.
    • Apache 서버 같은 경우 요청이 들어오면 connection(네트워크 통신에서 클라이언트와 서버 간의 연결)을 생성한다. 그래서 새로운 요청이 들어올 때마다 프로세스를 생성한다. 하지만 이 과정이 오래 걸리기 때문에 미리 만들어 놓은 프로세스를 할당한다. 만약 만들어 놓은 게 없는 경우에는 추가로 프로세스를 생성한다.
    • Apache 서버의 문제점은 요청이 많아져서 서버에 동시에 연결된 connection이 10000개 이상일 때 새로운 connection을 생성하지 못한다는 것이다.
    • Windows
    • Linux
    • macOS
    • ...
    • 클라이언트로부터 요청을 받았을 때 요청에 맞는 정적 파일을 응답해주는 HTTP Web Server로 활용되기도 하고, 또는 Reverse Proxy Server로 활용하여 WAS의 부하를 줄일 수 있는 로드밸런서 역할을 한다.
    • NGINX는 아파치 서버의 구조적 한계를 극복하려고 제작되었다.
    • 한 개 또는 고정된 프로세스만 생성하고, 여러 개의 Connection을 모두 Event-Handler(네트워크 이벤트 처리)를 통해 비동기 방식으로 처리한다. 그래서 동시 접속자 수가 많아져도 추가적인 생성 비용이 들지 않는다.
    • 가볍다.
    • 모듈이 다양 하지 않다.
    • Windows
    • Windows server
    • Microsoft에서 지원하는 웹 서버 소프트웨어이다.
    • 간편한 GUI를 지원한다.
    • FTP, SMTP, NNTP, HTTP/HTTPS포함한다.
    • Apache와 Nginx에 비해 느리다.
